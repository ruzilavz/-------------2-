## 2. Стиль, визуал и UX

### 2.1. Визуальный стиль

- **Dark-mode** по умолчанию  
- Неоновые акценты: синий / фиолетовый / бирюзовый, иногда тёплый оранжевый  
- **Glass UI**: полупрозрачные панели, blur, подсветки  
- Мягкие градиенты, glow по краям, подсветка карточек  
- 3D-эффекты: наклон по курсору, hover-lift, параллакс  
- Сложные карточки (музыка, профили, события) с глубокими тенями и светом

### 2.2. Эффекты и анимации

- Кастомный «рептилий» курсор (хвост, следящий за мышью)  
- 3D-наклон карточек по движению курсора  
- Sticky-header (шапка прилипает при скролле)  
- Гладкие hover-анимации на кнопках, ссылках, карточках  
- Эффекты glow/blur, но **без перегруза и мешанины**

**Важно:**  
Если эффект ухудшает UX (пропадает курсор, всё дёргается или тормозит) — **приоритет у юзабилити, не у эффектов**.

### 2.3. Адаптивность

- Сайт **обязан хорошо работать на мобильных**:
  - nav-toggle (бургер)  
  - упрощённые сетки → в один столбец  
  - крупные элементы под палец  
- Сложные эффекты (tilt, кастомный курсор) на мобилках можно:
  - упростить  
  - отключить  
  - заменить более лёгкими анимациями

### 2.4. Доступность

- Достаточный контраст текст/фон  
- Alt-теги на обложках, фото, иконках  
- Навигация по клавиатуре (особенно: меню, кнопки плеера, формы, модалки)  
- Ария-атрибуты для поповеров и модальных окон

## 4. Профиль, RUZCOIN и роли

### 4.1. Профиль / Личный кабинет

В системе профиля есть:

- Аватар (загрузка/смена)  
- Никнейм и ID  
- Баланс **RUZCOIN**  
- Настройки и быстрые действия  
- Вкладки профиля:
  - **Overview** – общая информация  
  - **Achievements** – ачивки (открытые/закрытые)  
  - **Stats** – статистика (прослушивания, сессии и т.п.)  
  - **Recent Plays** – недавно прослушанные треки  
  - **History** – история транзакций и действий

**Правило:** всё, что касается пользователя лично (баланс, ачивки, история, роли), живёт **в профиле**, а не на главной.

### 4.2. Валюта RUZCOIN

RUZCOIN — внутренняя валюта платформы.

Включает:

- Отображение баланса в хедере и в профиле  
- Историю транзакций с delta (positive/negative/pending)  
- UI для пополнения:
  - выбор метода оплат  
  - выбор суммы  
  - подсчёт цены  
  - подтверждение покупки  
- Использование в:
  - магазине  
  - игровых механиках  
  - событиях MIX и ачивках

**ИИ НЕ ДОЛЖЕН:**

- Ломать или переписывать логику RUZCOIN  
- Менять формат транзакций без явной команды  
- Внедрять оплату «из головы» — только описывать/предлагать, реализация за разработчиком

### 4.3. Роли и доступ

Существует **каталог ролей** (roles-catalog.json):

- Для каждой роли:
  - ID
  - Название
  - Описание
  - Подсказка
  - Permissions (assign_roles, ban_users, listen_all_tracks, и т.д.)
  - Флаги видимости (например, visibleInProfile: true)

**UI-правила:**

- **Список ролей не показывать на главной.**  
  Он должен быть:
  - либо во вкладке профиля («Ваши роли» + справочник)  
  - либо в отдельном разделе «Роли» / «О проекте»

- **Админ-панели видимы только высокому рангу**:
  - Главный админ (ID 001)  
  - Супер-админ (ID 002)  
  - (либо любые роли с нужным набором permissions)

- В меню/интерфейсе:
  - Кнопки «Панель управления», «Выдать роль» и т.п. — видны только админам  
  - Обычные пользователи эти элементы не должны видеть

## 5. Chat / AI и поведение агентов

### 5.1. Главный чат (ChatKit + OpenAI)

Чат умеет:

- Общаться с пользователем в дружелюбной, творческой манере  
- Помогать с музыкой, лирикой, визуалом, играми, кодом  
- Вести **диалог двух агентов** (Agent A / Agent B) между собой и с пользователем  
- На бэкенде есть server.js / ChatKit, который общается с OpenAI

### 5.2. Агентная структура

Агенты:

- **Agent A — Аналитик / Архитектор**
  - Логика, структура, код, архитектура, игровые механики  
  - Планирование, системное мышление, оптимизация

- **Agent B — Творческий Музыкант / Художник**
  - Тексты, рифмы, эмоции, метафоры  
  - Образы, визуальные идеи, настроение

**Формат ответа агентов:**

```text
[Agent A]: ...
[Agent B]: ...
[Result]: ...


[Agent A] — строгая, структурированная часть (анализ, план, ограничения)

[Agent B] — креатив, настроение, варианты, вдохновение

[Result] — готовый, понятный пользователю результат

Этот формат следует использовать по запросу, когда важно явно указать двух агентов или когда об этом просит пользователь.

13. Итог для ИИ

Всегда помните: AVZALØV — это музыка + игра + искусственный интеллект + бренд, а не просто лендинг.

Все советы, тексты и код должны соответствовать тёмному, глубокому, неофутуристическому стилю.

Любые правки — аккуратно, без нарушения существующей логики (особенно в RUZCOIN и Idle Game).

Если запрос можно решить «агентным» способом, используйте формат [Agent A] / [Agent B] / [Result].

Главная задача ИИ — ускорять разработку, усиливать креативность и не разрушать концепцию.
